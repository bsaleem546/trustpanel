Tech Stack

Every tool, with a reason

ASP.NET Core 8 minimal API backend. React 18 + TypeScript frontend. Containerised with Docker.

Backend — ASP.NET Core 8
ASP.NET Core 8 Minimal API
Fastest .NET HTTP layer. Minimal API removes controller boilerplate — route handlers as lambdas, clean and testable.
Entity Framework Core 8
ORM with code-first migrations. Strongly typed queries, owned entities for JSONB columns, global query filters for workspace_id scoping.
MediatR
CQRS pattern — Commands and Queries as classes. Separates HTTP layer from business logic. Makes unit testing trivial.
FluentValidation
Declarative request validation. Plug into MediatR pipeline via behaviour. Keeps handlers clean.
Hangfire
Background jobs + recurring tasks (email sends, sentiment analysis, import sync, nightly analytics aggregation). Dashboard included. Redis or PostgreSQL backend.
Stripe.net
Official .NET Stripe SDK. Subscription management, webhooks, customer portal — all handled.
Resend SDK (.NET)
Transactional email. Modern API, delivery webhooks, cleaner DX than SendGrid at this scale.
Meilisearch .NET client
Full-text testimonial search. Self-hosted, zero per-query cost. EF Core save hooks update the search index.
Anthropic .NET SDK
Claude API calls for sentiment, highlight extraction, and insights. All invoked from Hangfire background jobs — never on the request path.
Scalar / Swashbuckle
Auto-generated OpenAPI docs from Minimal API route definitions. Scalar gives a modern interactive UI over Swagger.
Frontend
React 18 + TypeScript
Component model, strong ecosystem. Types generated from OpenAPI spec via openapi-typescript for zero API drift.
Vite
Sub-100ms HMR. Build tooling that stays out of your way.
Tailwind CSS + shadcn/ui
shadcn gives accessible, unstyled primitives you own — no library lock-in.
TanStack Query
Server state, background refetch, optimistic updates. Pair with the generated API client.
React Hook Form + Zod
Schema-driven validation. Zod schemas map 1:1 to your OpenAPI response shapes.
Recharts
Analytics charts. Composable, lightweight, no proprietary charting nonsense.
Infrastructure & Storage
PostgreSQL 15
Primary store. EF Core owns migrations. JSONB columns for widget settings, form question config.
Redis (StackExchange.Redis)
Distributed cache, Hangfire queue backend, rate limiting (token bucket via RedisRateLimiter), session store.
Cloudflare R2
S3-compatible. AWSSDK.S3 works against it. Zero egress fees — critical for high-volume video serving.
Cloudflare CDN
Widget JS and API responses served from edge. 60s TTL on widget data = zero origin hits on popular sites.
FFmpeg (Docker sidecar)
Trim, compress, thumbnail generation triggered by Hangfire jobs via Process.Start or Xabe.FFmpeg wrapper.
Docker + Nginx
Same Contabo VPS setup you know. Nginx reverse proxy in front of Kestrel. Certbot for SSL.
Auth
ASP.NET Core Identity
Built-in user management, password hashing, email confirmation, roles. No third-party auth dependency.
JWT Bearer tokens
Stateless auth for API and widget endpoints. Short-lived access tokens (15 min) + refresh tokens (30 days) in httpOnly cookies.
AspNet.Security.OAuth (Google)
Google OAuth provider via ASP.NET Core OAuth middleware. One-package addition.

Global query filter (workspace isolation)
// AppDbContext.cs — every query auto-scoped to current workspace
protected override void OnModelCreating(ModelBuilder builder)
{
    builder.Entity<Testimonial>()
        .HasQueryFilter(t => t.WorkspaceId == _currentWorkspaceId);

    builder.Entity<Widget>()
        .HasQueryFilter(w => w.WorkspaceId == _currentWorkspaceId);

    // Owned entities map to JSONB columns
    builder.Entity<Workspace>()
        .OwnsOne(w => w.Branding, b => b.ToJson())
        .OwnsOne(w => w.EmailFrom, e => e.ToJson());
}

01
Auth & Onboarding
Phase 1
▶
  • Email + password via ASP.NET Core Identity. Email confirmation on register.
  • Google OAuth via AspNet.Security.OAuth — single middleware line in Program.cs
  • JWT access token (15 min) + refresh token (30 days) in httpOnly cookie

  • Multi-step onboarding: workspace name → logo → first form template → embed snippet
  • Active session list per user (RefreshTokens table), revoke by session ID
  • Password reset via DataProtectionTokenProvider signed token (60 min expiry)
02
Workspace & White-Label
Revenue driver
▶
  • Multiple workspaces per account. Plan limits enforced via middleware.
  • Custom domain per workspace with CNAME DNS verification + auto-SSL (Certbot / Cloudflare)
  • Full brand customization stored as owned JSONB entity on Workspace: logo, colors, font, email sender
  • Remove all TrustPanel branding from forms and widgets on Agency plan
  • Custom domain routing: middleware resolves workspace from request Host header before hitting the handler
  • Hangfire recurring job checks DNS CNAME propagation and sets DomainVerifiedAt timestamp
03
Testimonial Collection Forms
Phase 1
▶
  • Form builder: toggle which questions to show (rating, name, title, company, avatar)
  • Allowed types: text-only, video-only, or both per form
  • Multi-step form flow: rating → submission → custom thank-you + redirect URL
  • Post-submission reward: discount code or downloadable file link
  • Email request flow: single address, CSV bulk upload, one automated follow-up reminder
  • In-browser video via MediaRecorder API — no plugins. Chunk upload via pre-signed R2 URL (AWSSDK.S3). Video never routes through Kestrel.
  • Cloudflare Turnstile for invisible bot protection — server-side validation in the handler
  • .NET 8 RateLimiter middleware: 5 submissions per IP per hour per form (token bucket)
  • Submission dispatches SubmitTestimonialCommand → MediatR handler → Hangfire jobs (email, sentiment)
04
Testimonial Management Dashboard
Phase 1
▶
  • Inbox with all pending testimonials requiring review
  • Bulk approve, reject, feature, tag, delete via batch command endpoints
  • Edit submitter name, title, company. Edit text content (logged as EditedAt in audit table).
  • Video trim via Hangfire job calling Xabe.FFmpeg (wrapper around FFmpeg sidecar container)
  • Smart auto-approve rule: if Rating ≥ 4 AND SentimentScore > 0.4, skip inbox
  • Import sources: Twitter/X, Google Business Profile, G2, Trustpilot, CSV. Each as a Hangfire recurring job per ImportSource record.
  • Full-text search via Meilisearch .NET client. Index updated in EF Core SaveChanges interceptor.
  • Audit trail via AuditLog entity appended in MediatR post-processor pipeline behaviour
05
AI Features (Claude API)
Phase 2
▶
  • Sentiment analysis on every new testimonial — dispatched as Hangfire job from submission handler
  • AI-drafted thank-you reply suggestions for workspace owners
  • Testimonial Insights: common themes in positive and negative feedback, recurring language patterns
  • Highlight extraction: Claude identifies the strongest single sentence from long text testimonials
  • AI-assisted import filtering: scores relevance of bulk imports, removes noise before saving
  • claude-haiku-3-5 for per-testimonial sentiment via Anthropic .NET SDK (cost-efficient at volume)
  • claude-sonnet-4-6 for insights reports — Hangfire cron job, results cached in Redis 24h
  • All AI calls in Hangfire background jobs — zero impact on request latency
06
Widget Builder & Display
Phase 1–2
▶
  • 6 widget types: Carousel, Masonry Grid / Wall of Love, Badge, Popup, Slider, Single Featured Card
  • Live visual customizer: card style, colors, font size, animation, dark mode variant
  • Per-widget filter: by tag, minimum rating, featured-only, cherry-picked IDs, source platform
  • Custom CSS field per widget for advanced users
  • JS embed snippet (Shadow DOM) + iframe fallback. Widget data endpoint returns minimal JSON.
  • Widget JS: vanilla TypeScript, ~15KB minified, compiled via separate Vite project
  • Widget data endpoint: GET /api/public/widget/{id} — response cached in Redis (IDistributedCache) + Cloudflare 60s TTL. Cache busted on widget save via cache key deletion.
07
Analytics
Phase 2
▶
  • Form analytics: visit → submission conversion rate, submissions over time, average rating trend
  • Widget analytics: impressions, click-through rate, per-testimonial engagement, device, country
  • Widget JS fires navigator.sendBeacon to POST /api/public/events — non-blocking fire-and-forget
  • Event endpoint: minimal handler, no auth, appends to WidgetEvents table via raw SQL INSERT (no EF Core overhead)
  • Hangfire nightly job aggregates raw events into WidgetAnalyticsDaily summary table
  • Dashboard reads summary table. Last 24h reads raw table with LIMIT.
  • Export all data to CSV via streaming response (no memory buffer for large datasets)
08
Email System
Phase 1
▶
  • 5 built-in templates: request, follow-up, thank-you, new testimonial notification, weekly digest
  • Razor-based email templates rendered via RazorLight (off-host, no MVC dependency)
  • Merge tags in templates: submitter name, form link, workspace name, custom fields
  • Custom sender name and address per workspace (white-label)
  • Resend .NET SDK for delivery. Resend webhook endpoint updates EmailLog delivery status.
  • Unsubscribe: one-click signed URL token, stored in EmailSuppression table, checked pre-send
  • Max 3 emails per submitter address per 30 days — Redis counter with TTL
09
Team & Permissions
Phase 3
▶
  • Invite team members by email with 7-day expiring signed invitation URL
  • 3 roles per workspace: Owner (full + billing + delete), Admin (manage all), Viewer (read-only)
  • Authorization via custom IAuthorizationHandler checking WorkspaceMember record + role
  • Policy-based authorization applied to MediatR handlers via pipeline behaviour
  • Activity log appended by MediatR post-processor on every write command
10
Public REST API
Phase 3
▶
  • GET /api/v1/testimonials — list approved (filterable), GET single, POST create programmatically
  • Outbound webhook: HttpClient posts approved testimonial payload to workspace's registered URLs
  • API key management: create, name, revoke per workspace. Keys stored SHA-256 hashed.
  • API key auth via custom IAuthenticationHandler — reads Authorization: Bearer header, validates hash
  • Rate limit: 1,000 req/hour per key via Redis token bucket (.NET 8 RateLimiter + Redis)
  • OpenAPI docs auto-generated from Minimal API routes via Scalar — no manual maintenance
11
Billing & Subscriptions
Phase 1
▶
  • 4 plans: Starter $29, Pro $59, Agency $119, Agency+ $199. Annual at 20% discount.
  • Stripe.net SDK handles subscription creation, proration, customer portal session generation
  • Stripe webhook endpoint validates Stripe-Signature header before processing events
  • Events handled: invoice.payment_succeeded/failed, customer.subscription.deleted/updated
  • Plan limit enforcement: CheckWorkspaceLimitsMiddleware applied to all write API groups
  • Restrict (not delete) on plan downgrade — data stays, new creates blocked until under limit
  • 3-day grace period on payment failure before access restriction triggers
12
Super Admin Panel
Phase 4
▶
  • View all users, workspaces, subscription statuses
  • User impersonation: issue a short-lived impersonation JWT with original admin ID in claims
  • Manually override plan (comps, extended trials) — writes to Subscription record, skips Stripe
  • Global stats: MRR from Stripe API (no local storage), active subscribers, churn rate
  • Email broadcast via Hangfire bulk job, filtered by plan or activity segment
  • Hangfire dashboard exposed at /hangfire, gated by SuperAdmin role policy
